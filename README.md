# Codebase Copilot

Codebase Copilot is a local code-repository agent built with Python and C++. Python handles the workflow, prompting, and CLI orchestration. C++ handles the native vector search path through `pybind11`.

## What It Does

- scans and chunks a local repository
- builds a `metadata.json` index for retrieval
- answers codebase questions with grounded source chunks
- produces patch-style suggestions with file targets, reasons, and sketch diffs
- benchmarks Python brute-force retrieval against the C++ retriever

## Architecture

```text
User / CLI
    |
    v
python/main.py
    |
    +--> Repository Loader / Chunker / Index Builder
    |        |
    |        v
    |    metadata.json
    |
    +--> Hashing Embedder
    |
    +--> C++ VectorRetriever (pybind11)
    |        |
    |        v
    |    Top-K Chunks
    |
    +--> Prompt Builder / Local Synthesizers / OpenAI-compatible LLM
             |
             v
      Answer / Patch Suggestion / Benchmark Report
```

## CLI Quick Start

### Index a Repository

```powershell
python python/main.py index --repo . --output data/metadata.json
```

### Ask a Code Question

```powershell
python python/main.py ask "Where is the application entry point?" --index data/metadata.json --top-k 3
```

### Request a Patch Suggestion

```powershell
python python/main.py patch "How should I add input validation to the login flow?" --index data/metadata.json --answer-mode local --top-k 4
```

### Run the ReAct Agent

```powershell
python python/main.py agent "Where is the application entry point?" --index data/metadata.json --answer-mode local --max-steps 4
```

### Run the Benchmark

```powershell
python python/main.py benchmark
```

You can override the benchmark scale:

```powershell
python python/main.py benchmark --sizes 1000,10000,50000,100000 --dimension 64 --query-count 20 --top-k 5 --output data/day6_benchmark.md
```

## Optional LLM Backend

```powershell
$env:CODEBASE_COPILOT_LLM_API_KEY="your-api-key"
python python/main.py ask "Where is the application entry point?" --index data/metadata.json --answer-mode llm --llm-model qwen3.5-122b-a10b
python python/main.py patch "How should I add exception handling here?" --index data/metadata.json --answer-mode llm --llm-model qwen3.5-122b-a10b
```

If your provider is not Alibaba Cloud DashScope, also set `CODEBASE_COPILOT_LLM_BASE_URL` or pass `--llm-base-url`.

## Benchmark

The current benchmark report is generated by the CLI and saved to `data/day6_benchmark.md`.

Benchmark parameters used for the table below:

- synthetic random unit vectors
- dataset sizes: `1k`, `10k`, `50k`, `100k`
- dimension: `64`
- query count: `20`
- top-k: `5`
- top-k consistency check: first `5` queries

| Dataset | Python Avg (ms) | C++ Avg (ms) | Speedup | Top-K Match |
| ------: | --------------: | -----------: | ------: | :---------: |
| 1,000 | 10.939 | 0.034 | 324.13x | yes |
| 10,000 | 103.938 | 0.265 | 391.52x | yes |
| 50,000 | 648.134 | 3.697 | 175.31x | yes |
| 100,000 | 1230.405 | 6.051 | 203.34x | yes |

## Project Highlights

- native retrieval core implemented in C++ instead of calling an external vector database
- `pybind11` bridge that exposes the C++ retriever cleanly to Python
- grounded QA and patch suggestion flows that return file paths and source chunks
- quantitative benchmark showing the native retriever is materially faster than the Python brute-force baseline
